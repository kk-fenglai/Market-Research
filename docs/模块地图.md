# 模块地图（saas-starter）

把工程分成两层：**CORE 通用平台层（复用）** 与 **BUSINESS 业务示例层（DELF，可替换/可删）**。
本骨架由 `delf-B2-website` 抽取而来，已脱敏（无 `.env`、无真实密钥、无内部文档、无题库大文件）。

> **边界实现方式**：为避免移动文件破坏数十处相对 `require`，业务文件**仍按原目录平铺**在 `src/` 下，
> 模块边界用**注释围栏**（`↓↓↓ 业务示例 … ↑↑↑`）+ 本文件的**删除清单**标注。
> 想物理隔离时，可后续再按本表把 BUSINESS 文件挪进 `src/modules/business-delf/` 并修正引用。

## 删除业务清单（把骨架变成纯平台）

1. **后端 schema**：删 `backend/prisma/schema/90-business-delf.prisma`；
   在 `backend/prisma/schema/00-core.prisma` 的 `User` 里删「业务示例关系」整段（已用 `↓↓↓/↑↑↑` 框好）。
2. **后端装配**：`backend/src/index.js` 删 3 处业务围栏段（require / BUSINESS 挂载 / business worker + drain），
   以及 ADMIN 段里 `adminExams` 那一行。
3. **后端文件**：删上表 B1/B2 列出的 `routes/services/utils/constants/scripts` 业务文件。
4. **前端**：`frontend/src/App.tsx` 删业务 lazy import 段 + 业务路由段 + 后台 exams 3 行；删上表 B1/B2 前端文件。
5. **依赖/资源**：按需从两个 `package.json` 移除业务依赖；删 `backend/content`、`backend/assets`、`*.traineddata`（本骨架未带）。
6. **迁移**：`npx prisma migrate dev --name drop_business`（基于新 schema 生成去掉业务表的迁移）。

> 反向操作（换新业务）：照着 `90-business-delf.prisma` 的写法新建你的领域 schema 文件，
> 在 `index.js` 业务围栏处挂你的路由，在 `App.tsx` 业务围栏处加你的页面即可。
